事件监听和事件绑定

事件绑定 —— 让网页交互起来

重要性：★★★★★（实现动态交互的桥梁）

1. 什么是事件与事件监听

  • 事件：HTML 事件是发生在 HTML 元素上的“事情”。例如：按钮被点击、鼠标移动到元素上、按下键盘按键。
  • 事件监听：JavaScript 可以在事件被侦测到时 执行特定代码。

2. 两种核心的事件绑定方式

方式一：通过 HTML 标签中的事件属性绑定

  • 做法：在 HTML 标签中直接写入事件属性（如 onclick），并为其指定要执行的函数。
  • 注意：属性值里调用自定义函数时，**必须带括号 on()**。
  • 优缺点分析：
    • 优点：直观、简单。
    • 缺点：HTML 结构代码和 JS 逻辑代码严重耦合，在大型项目中极难维护，实际开发中不推荐。

方式二：通过 DOM 元素属性绑定（🌟 强烈推荐）

  • 做法：首先通过 DOM 抓取到 HTML 元素对象，然后将一个匿名函数赋值给该元素的事件属性。
  • 注意：赋值时，右边的匿名函数 function(){...}。
  • 优缺点分析：
    • 优点：实现了 HTML 结构与 JS 逻辑的完全解耦，代码清晰，符合现代前端规范。

常见事件

在 JavaScript Web 开发中，常见的事件主要可以分为 鼠标事件、键盘事件、表单事件 和 页面/窗口事件 四大类。

掌握这些事件是让网页产生动态交互的基础，以下是为你整理的常用事件全景指南：

一、 鼠标事件（Mouse Events）

触发时机：用户使用鼠标或触控板操作网页元素时。

  • onclick（🌟 最常用）：鼠标单击。
    • 应用场景：按钮提交、关闭弹窗、切换标签页（Tab）。
  • ondblclick：鼠标双击。
    • 应用场景：双击放大图片、双击编辑文本列表。
  • ononmouseenter / onmouseover：鼠标移入某个元素。
    • 应用场景：电商网站鼠标移入商品图时显示放大镜效果，或者下拉菜单滑出。
  • onmouseleave / onmouseout：鼠标移出某个元素。
    • 应用场景：隐藏刚刚滑出的下拉菜单。

  • onmousemove：鼠标在元素上移动（会连续不断地触发）。
    - 应用场景：网页拖拽功能、H5 游戏中的准星跟随、数据可视化图表（ECharts）的悬浮提示框。

二、 键盘事件（Keyboard Events）

触发时机：用户按下或释放键盘按键时。常用于需要输入或快捷键操作的场景。

  • onkeydown（🌟 最常用）：键盘按键被按下。

    • 应用场景：最经典的用途是监听 Enter（回车）键。用户在登录页面输完密码后，直接敲回车就能触发登录，无需手动去点登录按钮。
    • 小贴士：在事件函数中，可以通过 event.keyCode 或 event.key 来判断用户具体按下了哪一个键。

  • onkeyup：键盘按键被释放（弹起）。

    • 应用场景：实时字数统计。比如发微博或发帖时，用户每打完一个字弹起键盘，就实时计算并显示“还可以输入 xxx 字”。

三、 表单事件（Form Events）

触发时机：用户在填写表单、选择选项或提交输入框时，专门用来做表单验证和交互联动。

  • onfocus：元素获得焦点（光标点进去）。

    • 应用场景：输入框被点击时高亮显示边框，或者在输入框下方弹窗提示密码强度要求。

  • onblur（🌟 核心常考）：元素失去焦点（光标移出来）。

    • 应用场景：前端实时表单验证。当用户输完用户名，光标点到密码框的瞬间，触发用户名的 onblur 事件，立刻检查该用户名是否合法或是否已被注册，并直接在旁边标红报错（如图案）。

  • onchange：内容发生改变且失去焦点。

    • 应用场景：省市区三级联动下拉菜单。当用户把“省份”从“广东省”改成“北京市”时，触发 onchange，后面的“城市”下拉菜单自动跟着刷新。

  • oninput：内容正在发生实时改变（不需要失去焦点）。

    • 应用场景：搜索框的“搜索联想/智能推荐”功能。每打一个字母，立刻向后端发请求获取推荐关键词。

  • onsubmit：表单提交时（挂在 <form> 标签上）。

    • 应用场景：在数据真正发给后端之前做最后的总检查。如果验证不通过，可以通过 return false 拦截提交。

四、 页面与窗口事件（Page/Window Events）

触发时机：由整个浏览器窗口（window 对象）或网页生命周期触发。

  • onload：网页及所有资源（图片、样式表等）完全加载完毕。
    • 应用场景：代码初始化。有时候把 <script> 脚本写在 HTML 头部，由于此时下方的 HTML 标签还没加载出来，直接获取 DOM 会报错。如果包裹在 window.onload = function(){...} 内部，就能确保在页面完全加载后再执行 JS。
  • onresize：浏览器窗口大小发生改变。
    • 应用场景：响应式布局或图表自适应。当用户拉大/拉小浏览器窗口时，重新计算网页布局或让数据图表重新渲染铺满屏幕。
