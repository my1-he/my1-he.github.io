    BOM和DOM

    BOM（Browser Object Model）—— 浏览器对象模型

    重要性：★★★☆☆（了解核心对象及网页跳转即可）

    1. 概念

    将浏览器的各个组成部分封装为对象，允许 JavaScript 与浏览器进行“对话”。

    2. 核心组成对象

    BOM 包含五个核心大对象：

    1. Window：浏览器窗口对象（老大的位置，一切的基石）。
    2. Navigator：浏览器对象（包含浏览器版本、内核等信息）。
    3. Screen：屏幕对象（包含屏幕分辨率等信息）。
    4. History：历史记录对象（控制前进、后退）。
    5. Location：地址栏对象（最常用，需重点掌握）。

    3. Window 对象核心方法

    window 对象是全局对象，调用它的属性和方法时，**window. 可以省略**。

    • 弹窗方法：
      • alert()：显示带有一段消息和一个“确认”按钮的警告框。
      • confirm()：显示带有一段消息以及“确认”和“取消”按钮的对话框（返回 true 或 false）。
    • 定时器方法（高频实用）：
      • setInterval(fn, ms)：按照指定的周期（以毫秒计）循环调用函数。
      • setTimeout(fn, ms)：在指定的毫秒数后只调用一次函数（延时执行）。

    4. Location 对象（高频考点）

    • 介绍：地址栏对象。
    • 获取与使用：通过 window.location 获取，可省略写为 location。
    • 核心属性 href：设置或返回完整的 URL。常用于实现网页自动跳转。

    DOM（Document Object Model）—— 文档对象模型

    重要性：★★★★★（核心基础，操作网页元素的唯一方式）

    1. 概念与价值
    • 概念：DOM 是 W3C 的标准，它将标记语言（HTML/XML）的各个组成部分封装为对应的对象。
    • 核心价值：JavaScript 只有通过 DOM，才能够动态地对 HTML 进行操作（改变元素内容、改变 CSS 样式、对事件作出反应、添加/删除 HTML 元素）。

    1. DOM 的三大分类

    2. Core DOM：所有文档类型的标准模型（包含 Document, Element, Attribute, Text, Comment 等对象）。

    3. XML DOM：XML 文档的标准模型。

    4. HTML DOM：专门针对 HTML 文档的标准模型（在 Core DOM 基础上进行了扩展，如 Image, Button 等专门标签对象）。

    5. DOM 树模型结构

    当浏览器加载 HTML 页面时，会将其解析为一棵“DOM 树”：

    • 整个文档是一个 Document 对象（可以通过 window.document 或直接用 document 获取）。
    • 所有的 HTML 标签都是 Element 对象。
