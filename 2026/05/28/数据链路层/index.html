数据链路层

发表于 2026-05-28 分类于 计算机网络
计网的知识点也太多了

一、 信道类型与基本概念

  • 信道类型：
    • 点对点信道： 一对一通信（如 PPP 协议）。
    • 广播信道： 一对多通信（如以太网，需局域网协议协调）。
  • 数据链路层的三个基本问题：
    1. 封装成帧 (Framing)： 在一段数据的前后分别添加首部和尾部，以此来构成一个帧。首部和尾部的一个重要作用就是进行帧定界（确定帧的界限）。
    2. 透明传输 (Transparent Transmission)：
      • 定义： 无论发送什么样的比特组合，数据都能够原样无误地通过数据链路层。也就是说，链路层对于上层交付的数据而言是“透明”的，就像没有阻碍一样。
      • 问题所在： 当数据中偶然出现了与“控制字符”（如帧定界符）相同的二进制比特时，链路层可能会产生误判。

​ 3. 差错检测

二、 封装成帧的控制机制（字节填充与零比特填充）

根据具体协议的不同，解决透明传输和控制字符冲突的做法主要有以下两种：

1. 字节填充 (Byte Stuffing) — 面向字节的链路（如 PPP 协议）

  • 标志字段 (Flag)： 规定特定比特（如 0x7E）为帧的边界。
  • 转义字符： 引入转义字符（如 0x7D）。
  • 具体做法规则：
    • 若数据中出现与标志字段相同的 0x7E $\rightarrow$ 转换为两字节序列 (0x7D, 0x5E)。
    • 若数据中偶然出现转义字符本身 0x7D $\rightarrow$ 转换为两字节序列 (0x7D, 0x5D)。

2. 零比特填充 (Zero-bit Stuffing) — 面向比特的链路

  • 标志字段： 同样以特定比特流（例如连续的 6 个 1：01111110）作为帧边界。
  • 具体做法规则：
    • 发送端： 在发送数据时，只要**连续发现 5 个连续的 1，就必须在其后面强制填入 1 个 0**。
    • 接收端： 在收到连续 5 个 1 后，若紧跟的是 0，则自动把该 0 删除，以此恢复出最原始的数据，从而保证透明传输。

三、 差错检测：循环冗余检验 (CRC)

数据链路层使用可靠的数学手段来检测传输中是否发生了“比特差错”。

1. 模 2 算法核心

  • 模 2 加减法： 本质就是异或 (XOR) 运算。即相同的比特相加减得 0，不同的比特相加减得 1。

    $1 \oplus 1 = 0$, $0 \oplus 0 = 0$, $1 \oplus 0 = 1$, $0 \oplus 1 = 1$

  • 模 2 除法： 纯粹按位异或。在除法求余数的每一步中，只要当前余数的最高位是 1，商就是 1，并用除数进行异或；最高位是 0，商就是 0。它不向前借位。

2. 发送端的冗余码（FCS）计算

  1. 设待发送的数据为 $M$（共 $k$ 位），约定的除数（生成多项式）为 $P$（共 $n+1$ 位）。
  2. 在数据 $M$ 后面添加 $n$ 个 0，使其变成 $k+n$ 位的全新数据 $\rightarrow M \cdot 2^n$。
  3. 用此全新数据与除数 $P$ 进行模 2 除法运算，得到一个 $n$ 位的余数 $R$（该余数 $R$ 即为 FCS 帧检验序列）。
  4. 将计算得到的 $R$ 直接拼接在原始数据 $M$ 后面，组成真正的传输帧发送出去。

3. 接收端 CRC 检验流程

  • 接收端收到数据帧后，将接收到的整个数据（含 $M+R$）拿去**模 2 除以同样的除数 $P$**。
  • 结果评判：
    • 若余数 $R = 0$：证明传输过程无错，予以接收。
    • 若余数 $R \neq 0$：证明传输过程有错，直接丢弃该帧。

四、 局域网总线协议：CSMA/CD 协议

在同一个时间段内，广播总线只允许一台计算机发送数据，否则会产生波形叠加（冲突）。为此引入了 CSMA/CD（载波监听多点接入/碰撞检测） 协议。

  • 多点接入： 许多计算机连接在同一根总线上，共享传输介质。
  • 载波监听： “边发送边监听”。在发送前及发送过程中，不间断地检测总线上有没有其他计算机在发送信号。
  • 碰撞检测： “边发送边检测”。一旦监听到总线上的电压摆幅超过了设定阈值，即说明此时至少有两台计算机同时发包，发生了碰撞（冲突）。

1. 碰撞的重要特性与限制

  • 电磁波传播延时： 电磁波在 $1 \text{ km}$ 光缆/电缆中的传播时延大约为 **$5\ \mu\text{s}$**。
  • 半双工限制： 因为要时刻预防和检测碰撞，使用 CSMA/CD 协议的局域网不可能同时进行发送和接收，因此它只能实现半双工通信。
  • 集线器 (Hub) 的物理特性：
    • 基带信号： 物理层传输的是由计算机直接发出的、未经过调制的原始电信号。
    • 无源电缆： 传输线缆的内部没有任何有源电子器件去放大或处理信号，完全依赖两端设备的自驱力。

2. 争用期与退避算法

  • 争用期 (Collision Window)：

    • 定义为信号在总线两端往返的时间 $2\tau$（$\tau$ 为单程传播时延）。

    • 最后 1 个 bit 传输到目的地前，都有可能遭遇碰撞。

    • 成功发送一个帧需要占用时间 $T_0$。定义参数：

      $$\alpha = \frac{\tau}{T_0}$$

  • 退避算法（截断二进制指数退避算法）：

    • 当检测到碰撞后，计算机必须停止发送，并推迟一段随机的时间后再重新尝试发送。
    • 信道利用率判定： 先经过装有若干个争用期 $2\tau$ 的冲突竞争阶段，最终 + 发送成功。

1. 确定基本退避时间（争用期 $2\tau$）

  • 基本退避时间就是争用期（碰撞窗口），其具体时间为 **$51.2\ \mu\text{s}$**。
  • 在 $10\text{ Mbit/s}$ 的传统以太网中，争用期内可发送 $512\text{ bit}$（即 $64\text{ 字节}$）。因此，争用期也经常直接被称为 **$512\text{ 比特时间}$**。

2. 计算控制参数 $k$

参数 $k$ 决定了随机数可选范围的大小。计算公式为：

$$k = \text{Min}[\text{重传次数}, 10]$$

  • 规则： * 当重传次数不超过 10 次时， $k$ 等于实际的重传次数。
    • 当重传次数超过 10 次时， $k$ 不再增大，一直等于 10。

3. 选取随机数 $r$ 并计算退避时间

  • 从离散的整数集合 $[0, 1, \cdots, (2^k - 1)]$ 中随机抽取一个整数，记为 **$r$**。

  • 重传应推迟的时间（即退避时间）为：

    $$r \times \text{争用期 (即 } r \times 51.2\ \mu\text{s} \text{ 或 } r \times 512\text{ 比特时间)}$$

4. 异常处理：放弃该帧

  • 如果连续重传 16 次 仍不能成功，说明网络中同时打算发送数据的站太多，导致连续冲突过于严重。
  • 此时接收端将丢弃该帧，并向高层协议报告。

实例演练（以具体碰撞次数为例）

  • 第 1 次重传：
    • $k = 1$，随机数 $r$ 从集合 ${0, 1}$ 中随机选取。
    • 推迟的时间可选：$0$ 或 $2\tau$。
  • 第 2 次重传：
    • $k = 2$，随机数 $r$ 从集合 ${0, 1, 2, 3}$ 中随机选取。
    • 推迟的时间可选：$0, 2\tau, 4\tau$ 或 $6\tau$。
  • 第 3 次重传：
    • $k = 3$，随机数 $r$ 从集合 ${0, 1, 2, 3, 4, 5, 6, 7}$ 中随机选取。
    • 推迟的时间依此类推。

五、 以太网 MAC 层与硬件地址

1. 适配器 (网卡)

  • 计算机与外界局域网连接的接口设备。
  • 内置有专职的处理硬件和存储器（如 RAM 和 ROM）。
  • 核心功能： 计算机硬件地址（MAC 地址）就固化在网卡的 ROM 中；网卡负责把网络层的 IP 数据报封装成 MAC 帧，或者将收到的 MAC 帧拆封还原。

💡 核心概念辨析：

  • IP 地址： 属于网络层，是逻辑上的虚拟地址（用于全局路由寻址）。
  • MAC 地址： 属于数据链路层，是物理硬件的固有地址（局域网内精准识别）。

2. 局域网拓扑演变

  • 使用集线器的传统局域网虽然在物理上呈现星形拓扑，但其逻辑上依然是一根总线。所有主机共享同一带宽，仍需严格使用 CSMA/CD 协议。

3. 以太网 V2 的 MAC 帧格式详解 (对照图 3-22 结构)

根据教材插图 1000055077.jpg 的严格定义，一个完整的以太网 MAC 帧由 5 个字段 构成：

  • 目的地址 (6 字节)： 接收端的硬件 MAC 地址。

  • 源地址 (6 字节)： 发送端的硬件 MAC 地址。

  • 类型 (2 字节)： 用来标志上一层使用的是什么协议，以便把收到的 MAC 帧数据准确上交给对应的上层协议。

    • 例如： 当该字段的值为 0x0800 时，表示上层使用的是 IP 数据报。

  • 数据字段 (46 ~ 1500 字节)： 帧的有效载荷（即上层的 IP 数据报）。

    • 为什么数据字段最小是 46 字节？ 以太网规定最短有效帧长为 64 字节。减去首部和尾部共 18 字节（$6+6+2+4 = 18$），即得出数据字段的最小长度为：

      $$64 - 18 = 46 \text{ 字节}$$

  • FCS 字段 (4 字节)： 帧检验序列，用于存放上一节提到的 CRC 校验码。

⚠️ 物理层附加元素（插入部分）

在 MAC 帧被传送到物理层准备在网线上发射时，物理层会在帧的前面插入 8 个字节 的前导数据：

  • 前同步码 (7 字节)： 复杂的比特流（10101010...），用于使接收端的时钟与发送端达到同步。
  • 帧开始定界符 (1 字节)： 比特流为 10101011，最后的两个连续 1 明确提示接收端：“真正的 MAC 帧从下一个比特正式开始！”。

六、 数据链路层扩展与现代以太网

随着技术演进，现代局域网的形态发生了根本性改变：

1. 以太网交换机 (Switch) 扩展

  • 网桥与交换机： 现代局域网采用交换机作为核心连接设备（以太网交换机实质上是一个多端口的网桥）。
  • 不再有碰撞问题： 交换机具有独占带宽和并行转发能力，并且主机与交换机之间采用全双工方式工作。因此，在现代由交换机组建的以太网中，不再使用 CSMA/CD 协议。

2. 虚拟局域网 (VLAN)

  • 通过局域网内的软件配置，将物理上的局域网设备按需逻辑划分成不同的“虚拟子网”，从而能够有效隔离广播域，提高安全性和网络效率。

3. 以太网宽带接入：PPPoE 协议

  • PPPoE (PPP over Ethernet)： 意思是“在以太网上传输 PPP 协议”。
  • 应用场景： 它把点对点协议（PPP）封装到以太网架构中。我们常见的家庭光纤宽带拨号上网，运营商就是利用 PPPoE 来对用户进行身份认证、计费和动态分配 IP 地址的。
