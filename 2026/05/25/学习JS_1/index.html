JS基本语法

一、 JS 引入与基本语法

1. 引入方式

  • 内部脚本： 使用 <script> 标签包裹。

    HTML

  • 外部脚本： 通过 <script src="file.js"></script> 引入。

    💡 最佳实践： 现代开发中，通常将 <script> 标签放在 <body> 的最底部，或者加上 defer / async 属性，防止 JS 脚本阻塞 HTML 页面的解析和渲染。

2. 变量 (Variables)

  • var 关键字： * 特性： 作用域较大（函数作用域或全局作用域），存在**变量提升 (Hoisting)**。
    • 缺点： 允许重复定义，容易污染全局变量或引发难以排查的 Bug。
  • let 与 const：
    • let： 声明块级作用域（Block Scope）的变量，不能重复声明，没有变量提升，是现代 JS 代替 var 的首选。
    • const： 声明块级作用域的常量。一旦赋值就不能再重新赋值（如果是对象或数组，其内部的属性或元素是可以修改的，但引用地址不能变）。

. 变量声明声明对比（var vs let vs const）

特性 var let const
作用域 函数作用域 / 全局作用域 块级作用域（{} 内有效） 块级作用域（{} 内有效）
变量提升 会提升（值为 undefined） 不会（存在暂时性死区） 不会
重复声明 允许 不允许 不允许
重新赋值 允许 允许 不允许（常量）

3. 数据类型 (Data Types)

  • 原始类型 (Primitive Types)：

    • number（数字，包含整数和浮点数）
    • string（字符串）
    • boolean（布尔值：true / false）
    • null（空值，表示一个“空”的对象引用）
    • undefined（未定义，声明了变量但未赋值时的默认值）
    • (拓展补充：ES6+ 还增加了 symbol 和 bigint)

  • 类型检测： 使用 typeof 运算符。

    ⚠️ 注意一个著名的 JS 历史 Bug： typeof null 的返回值是 "object"，而不是 "null"。

② 引用类型 (Reference Types)

  • Object、Array、Function 等。

4. 全等运算符 ===

  • ==（相等）：比较时会进行隐式类型转换。
  • ===（全等）：严格比较，不进行类型转换。只有当两边的值和类型都相同时才返回 true。
